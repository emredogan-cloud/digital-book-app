# Play Console — Internal Testing Upload Checklist (SUB-PR 1.3)

The signed `.aab` for upload is at:
```
android/app/build/outputs/bundle/release/app-release.aab     (~37 MB)
```
It's **gitignored** and regenerated by:
```bash
npm run cap:sync   # vite build && cap sync
cd android && ./gradlew bundleRelease
```
(Re-bump `versionCode` and `versionName` in `android/app/build.gradle` between uploads.)

---

## A. One-time setup (skip if already done)
1. Sign in to <https://play.google.com/console/>.
2. Pay the one-time **$25** developer registration fee.
3. Complete the developer profile (legal name, address, contact email, phone).

## B. Create the app
1. **All apps → Create app**.
2. **App name:** `Yaşayan Kütüphane` · **Default language:** Turkish (`tr-TR`) · **App / Game:** App · **Free / Paid:** Free.
3. Accept declarations → **Create app**.

## C. Store listing — paste from `STORE_LISTING.md`
1. Left nav → **Grow → Store presence → Main store listing**.
2. Paste the **TR** strings (app name, short, full).
3. **+ Add language → English (United States)** and paste the **EN** strings.
4. Upload graphic assets (placeholders OK for Internal Testing — see `STORE_LISTING.md` table).
5. **Category:** Books & Reference. **Save**.

## D. App content (compliance — left nav → Policy → App content)
- **Privacy Policy URL** — host the policy at a stable URL (e.g. a GitHub Pages page) and paste it.
- **App access** — full app, no restrictions.
- **Ads** — No.
- **Content rating** — fill the IARC questionnaire (Books & Reference, text-only, no explicit content → likely Teen).
- **Target audience** — 16+ (matches the literary/cyber-noir tone).
- **Data Safety** — declare *None* per `STORE_LISTING.md` (update in Phase 2 when analytics ships).
- **News app** — No. **COVID-19 contact tracing** — No.

## E. Internal Testing release
1. Left nav → **Release → Testing → Internal testing → Create new release**.
2. **App bundle:** upload `android/app/build/outputs/bundle/release/app-release.aab`. Play will scan it.
3. **Release name:** `1.0.0 (versionCode 1) — SUB-PR 1.3 internal`.
4. **Release notes:** copy a 3-line summary from the full description.
5. **Save → Review release → Start rollout to Internal testing.**

## F. Add internal testers
1. Same screen → **Testers** tab → **Create email list**.
2. Add your own Google account email (and any colleagues). Save.
3. Copy the **opt-in URL** Play generates — that's the install link for testers.

## G. Smoke test on a real device
1. On the Android device, open the opt-in URL in Chrome; accept testing.
2. Install via the Play Store.
3. Confirm:
   - App icon shows the Living Library mark.
   - App launches into the bookshelf (`Yaşayan Kütüphane`).
   - Every one of the 6 books opens and reads.
   - Run the `RUN.md` per-book checklist (smoke from SUB-PR 0.4) at least once.

## H. Iteration loop for the next signed build
```bash
# bump versionCode + versionName in android/app/build.gradle
npm run cap:sync
cd android && ./gradlew bundleRelease
# upload the new .aab to the same Internal Testing track.
```

## I. Promotion prerequisites (later, not now)
Before promoting to **Closed / Production**:
- Privacy policy URL is live and stable.
- **Data Safety** updated when Phase 2 ships PostHog/Crashlytics.
- **Sentry DSN populated** (`VITE_SENTRY_DSN` in `.env`) — current build has Sentry wired but disabled.
- The `RUN.md` per-book manual checklist (SUB-PR 0.4) executed on ≥2 OEM devices (SUB-PR 1.4).
- Privacy declarations match what the app actually does at runtime.

---

> The **keystore** at `android/app/release.keystore` is the app's signing identity for its whole life.
> **Back it up in ≥2 locations off this machine** (password manager + offline drive) before pushing
> any signed build publicly. Losing it = you can never update the app under this signature again.
