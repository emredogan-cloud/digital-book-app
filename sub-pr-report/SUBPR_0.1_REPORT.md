## What changed

**Context — re-homing (commit `ffa6344`, prerequisite, not a 0.1 deliverable):**
Per direction, the build was moved into `digital-book-app/`. The 6 books were copied in from the
**untouched** `MY-DİGİTAL-BOOK/` source; each book's nested `.git` repo was stripped so files track
as plain content in this monorepo — **no engine/content byte changed (Motor LOCK preserved)**. The
project constitution (`CLAUDE.md`, `.claude/settings.json` + `hooks/`, `memory/PAST_DECISIONS.md`,
`memory/USER_PROFILE.md`) was established here.

**SUB-PR 0.1 scaffold (commit 2):**
- `.gitignore` — ignores dependencies, native build output, logs, **signing keystores**, env files, editor/OS cruft, and the personal `.claude/settings.local.json` (the shared `settings.json` stays tracked).
- `package.json` — `living-library` private package; `serve` launches a local static server over `www/`; `build`/`lint`/`test` are placeholders wired up in SUB-PR 1.1.
- `package-lock.json` — generated by `npm install`.
- `www/shared/ll-namespace.js` — reserves `window.LL` (the single global contract between the frozen engine and the shell); no behavior yet.
- `www/books/.gitkeep` — keeps the empty consolidation target tracked until SUB-PR 0.2.
